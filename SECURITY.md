Security Policy

Reporting a Vulnerability

If you discover a security vulnerability in any Nivotics Lab repository, please do not open a public issue. Public disclosure before a fix is in place puts other users at risk.
Instead, report it privately by emailing us at: nivoticslab@gmail.com

Use the subject line: [SECURITY] Vulnerability in [repository name]

What to Include in Your Report
- The repository and file(s) affected
- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any suggested fix or mitigation, if you have one

What to Expect
- Acknowledgement of your report within 72 hours
- An initial assessment within 7 days
- A fix or mitigation plan communicated to you before any public disclosure
- Credit in the release notes if you would like it

Supported Versions

Security fixes are applied to the latest stable release of each project. Older versions are not actively patched unless the severity warrants it.


We take security seriously and appreciate responsible disclosure. Thank you for helping keep Nivotics Lab tools safe for everyone.
